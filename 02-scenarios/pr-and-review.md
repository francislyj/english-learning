# 场景：PR / Code Review

> 异步写 PR 评论你大概没问题；卡的是**口头 review、视频里实时表达立场**。
> 这里偏「能说出口」的口语版，不是书面版。

## 解释你的改动

| 中文 | English chunk |
| --- | --- |
| 我来过一遍这个改动 | Let me walk you through this change. |
| 这里的思路是 ___ | The idea here is ___. |
| 我这么做是因为 ___ | I did it this way because ___. |
| 这部分还在改，先别管 | This part's still a work in progress. |

## 请对方讲解 / 没听懂

| 中文 | English chunk | 情绪 |
| --- | --- | --- |
| 你能带我过一遍这块吗？ | Could you walk me through this part? | 🤔 |
| 这块逻辑我没太跟上 | I'm not quite following the logic here. | 😯 |
| 你能再说一遍吗？ | Could you say that again? | 🙂 |
| 你说的「___」是指？ | When you say "___", do you mean…? | 🤔 |

## 礼貌反对（你最该练的——既不软也不冲）

| 强度 | English chunk |
| --- | --- |
| 软（留余地） | I'm not sure about this one… / I have a slight concern here. |
| 中（明确提出） | I'd push back on this a bit. / I'd lean the other way here. |
| 直（明确反对） | I don't think this is the right approach. |

| 中文 | English chunk |
| --- | --- |
| 我懂你的意思，但是 ___ | I see what you mean, but ___. |
| 这样会不会有 ___ 的风险？ | Wouldn't this risk ___? |
| 我们要不考虑一下 ___？ | What if we considered ___? |

## 同意 / 收尾

| 中文 | English chunk |
| --- | --- |
| 有道理，就这么办 | Fair point, let's go with that. |
| 这个我后面跟进 | I'll follow up on that. |
| 其他看着都挺好 | The rest looks good to me. (LGTM) |

---

## 独白题目（录音用）

1. 60 秒：向同事口头解释你这个 PR 改了什么、为什么这么改。
2. 45 秒：你不同意 reviewer 的一个建议，礼貌地说出你的不同看法 + 给理由。
3. 30 秒：reviewer 说了一段你没听懂的话，请他放慢/重讲，并确认你的理解。
